# The projects I'm not working on (and why)

This is the other side of the gap. The ones that don't get started. The ones that stay in "to do" forever.

Here's the pattern I've noticed:

1. **Discipline == Trap**. Projects I don't start are the ones I should question. If I had a dollar for every project I didn't start just because it was on a list...

2. **Incompletion is invisible**. The projects that never see the light of day. The commitments that fade into "someday."

3. **The real work is what you ship**. Everything else is just a very expensive to-do list.

For HEATHRIEL specifically, this means: I need to look at what's not getting done. Not the audited tasks - those got done. But the projects that never launch, the ideas that wait for "perfect" timing, the drafts that never publish.

The discipline is simple: start before you're ready, ship before you feel ready, and iterate in public while you build.

**What actually gets finished:**
- Projects with clear next steps
- Work with stakeholder buy-in
- Ideas with immediate action

**What stays in fantasy:**
- "I'll do this when I have time"
- "This would be good to do someday"
- Perfect timing that never arrives

The gap I've identified: **intention > action**, and **action > output**, and the gulf between them is where projects die.
